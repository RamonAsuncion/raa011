DMA

Direct Access Memory

Direct Access Memory (DMA) is a method of transferring data from one place to another without involving the CPU. It is used to transfer data between the I/O device and memory. It is faster than interrupt driven I/O since the CPU is not involved in the data transfer. The CPU sets up the DMA controller and then it transfers the data directly to the memory. The CPU is free to do other work while the data transfer is in progress.

How it Works

DMA is a method of data transfer that allows the CPU to work on other tasks while the data transfer is in progress. The DMA controller is a separate chip that is connected to the system bus. When a DMA transfer is initiated, the DMA controller takes control of the system bus and transfers the data directly to the memory. The CPU is not involved in the data transfer. It is free to do other work while the data transfer is in progress. When the transfer is complete, the DMA controller releases control of the system bus and the CPU can resume its work.

Advantages

  • It is faster than interrupt driven I/O since the CPU is not involved in the data transfer.
  • It allows the CPU to work on other tasks while the data transfer is in progress.
  • It is more efficient than interrupt driven I/O since the CPU is not interrupted.

Disadvantages

  • It requires additional hardware (DMA controller).
  • It requires additional programming (DMA controller).
  • It requires additional memory (DMA controller).
